这里有一些很好的答案，他们都有一个共同点：测试。您需要经常测试并经常测试，最佳方法是自动化测试并将测试集成到您的构建链中。一旦完成每次代码被推到您的repo，它将被构建和测试。如果由于改变而被打破的事情，那么您的开发人员就会立即通知问题 - 而他们实施的解决方案仍然是新的。这允许他们快速且容易地发现和解决问题，意味着更少的时间和金钱。
