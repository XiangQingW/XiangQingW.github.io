首先，它被编制而不是解释。所以执行过程非常直接。此外，编译器已经调整并重新设计了几十年，以产生越来越更好的代码。
其次，C ++不需要实现关注未定义的行为。例如，编译器可以假设阵列的所有访问都在界限内;如果他们不是，无论发生什么，都会发生，编译器不需要。这与其他编译的编程语言完全不同，在这方面提供了一些安全性，但由于它们在生成的代码上强加了开销（必须检查索引界限的阵列的可能性）。阵列边界不是唯一的这样的情况：无需在执行时间内检查整数溢出，无效指针Dereference等。
第三，C ++提供了抽象机制，允许在库中包装复杂的工业强度算法，该算法在图书馆中添加了非常少的开支到直接代码扩展。多年来已经发布了成千上万的这些图书馆，因此应用程序通常可以快速部署调谐算法，以实现具有近最优机器效率的预期效果。
