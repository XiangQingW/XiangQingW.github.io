在20世纪80年代初的某些时候，Unix被描述为（可能是Dennis Richie）作为专业的新手漠不关心。这几乎存在。
在本发明的窗口系统之前，UNIX字面意思是终端 - 而不是终端流程 - 连接到具有实际电线的计算机系统，而不是管道。命令行是所有的。做任何事情，你通过命令行完成了它。
你认为这会令人生畏，没有？嗯……考虑一下：UNIX系统的第一个非技术用户是来自贝尔实验室专利部门的三个打字员，他们使用UNIX，ED编辑器和（上帝帮助我们全部）TROFF键入，然后排版专利申请。所有培训都很少。 1971年。在你的父母出生之前，最有可能。
1983年，在赖斯大学，我们将所有涉及计算机到UNIX的本科教育。 24行乘80个字符终端。命令行。而孩子们喜欢它 - 除了编程外，他们写了纸张，接触到早期的即时消息，各种各样的东西。没有gui。
事情是…… Unix命令行非常非常简洁 - 它旨在几乎看不见的是技术人员，零摩擦。对你来说，没有通过那个镜头看它，它几乎不透明，但相信我这是一个偏光镜头，当你转移你的观点时，透明度确实变得非常尖锐。
回到1999年，尼尔斯蒂芬森（是的，那个尼尔Stephenson）在一开始时写了一篇呼叫的文章，是命令行。虽然仍然可以作为平装，但它也可以作为PDF提供。
